## Bill relationships scraping — replaced chromote with static HTML

### Problem
`scrape_bill_relationships()` used `read_html_live()` (chromote/headless browser) on the BillBook page because the related bills section loads via JavaScript. This was slow, memory-intensive, and required chromium on the runner.

### Solution
Switched to the bill history page, which renders related bills in static HTML:
```
https://www.legis.iowa.gov/legislation/billTracking/billHistory?billName=HF%202511&ga=91
```
The "All Related Bills to Selected Bill" section contains `<a>` tags with hrefs like `/legislation/BillBook?ba=HF 2158&ga=91`. Bill numbers are parsed from the `ba=` query parameter.

URL construction: insert a space between letters and digits (`"HF2511"` → `"HF 2511"`), then `utils::URLencode()`.

XPath used to scope to the related bills section:
```
//text()[contains(., 'All Related Bills to Selected Bill')]/following::a[contains(@href, 'BillBook')]
```
